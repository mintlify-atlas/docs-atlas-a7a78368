## Quick Start

If you just want to build the app without setting up the backend, follow these steps:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ivansaul/cheat-sheets.git
    cd cheat-sheets
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    flutter pub get
    ```
  </Step>
  
  <Step title="Generate code">
    ```bash
    dart run build_runner build
    ```
  </Step>
  
  <Step title="Run the app">
    ```bash
    flutter run --release
    ```
  </Step>
</Steps>

<Note>
  The quick start method connects to the production API. For full local development with a local backend, see the [Full Development Setup](#full-development-setup) section below.
</Note>

## Full Development Setup

For complete local development including the backend API, follow these comprehensive instructions.

### Backend Setup

The app requires a backend API to serve cheat sheet content. Set up the backend first:

<Steps>
  <Step title="Clone the worker repository">
    ```bash
    git clone https://github.com/ivansaul/devstack-worker.git
    cd devstack-worker
    ```
  </Step>
  
  <Step title="Run the seed script">
    This creates and populates the database:
    ```bash
    cargo run -p cheatsheets
    ```
  </Step>
  
  <Step title="Initialize the database">
    ```bash
    npx wrangler d1 execute CHEATSHEETS --local --file=./schema.sql
    ```
  </Step>
  
  <Step title="Start the development server">
    ```bash
    npx wrangler dev --ip 0.0.0.0 --port 8787
    ```
    
    The backend API will be available at `http://localhost:8787`
  </Step>
</Steps>

### Flutter App Setup

With the backend running, now set up the Flutter app:

<Steps>
  <Step title="Clone the app repository">
    ```bash
    git clone https://github.com/ivansaul/cheat-sheets.git
    cd cheat-sheets
    ```
  </Step>
  
  <Step title="Configure environment variables">
    Create `.env` and `.env_debug` files in the root directory:
    
    ```bash .env
    API_CHEATSHEET_BASE_URL=http://localhost:8787/api
    API_CHEATSHEET_LIST_ENDPOINT=/cheatsheets
    API_CHEATSHEET_DETAIL_ENDPOINT=/cheatsheets/:id
    API_COUPON_BASE_URL=https://cdn.real.discount/api
    API_COUPON_LIST_ENDPOINT=/courses
    API_COUPON_ENDPOINT=/courses/slug/:slug
    ```
    
    <Note>
      **For Android Emulator:** Replace `localhost` with your computer's IP address (e.g., `http://192.168.1.100:8787/api`)
    </Note>
  </Step>
  
  <Step title="Install Flutter dependencies">
    ```bash
    flutter pub get
    ```
  </Step>
  
  <Step title="Generate code with watch mode">
    Run the build runner in watch mode to automatically regenerate code on changes:
    ```bash
    dart run build_runner watch -d
    ```
    
    Keep this running in a separate terminal window.
  </Step>
  
  <Step title="Run the app">
    In a new terminal window:
    ```bash
    flutter run
    ```
  </Step>
</Steps>

## Code Generation

The project uses code generation for various features. Here are the key commands:

<Tabs>
  <Tab title="Build Once">
    Generate code once:
    ```bash
    dart run build_runner build
    ```
  </Tab>
  
  <Tab title="Watch Mode">
    Automatically regenerate on file changes:
    ```bash
    dart run build_runner watch -d
    ```
  </Tab>
  
  <Tab title="Clean Build">
    Delete previous generated files and rebuild:
    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```
  </Tab>
</Tabs>

<Note>
  The project uses build_runner for generating code from annotations including:
  - Riverpod providers (`@riverpod`)
  - JSON serialization (`@JsonSerializable`)
  - Freezed models (`@freezed`)
  - Environment variables (`@Envied`)
  - Go Router routes (`@TypedGoRoute`)
</Note>

## Development Tips

  **Hot Reload:** Press `r` in the terminal running `flutter run` to hot reload changes without restarting the app.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build runner fails">
    Try cleaning the build cache:
    ```bash
    flutter clean
    flutter pub get
    dart run build_runner build --delete-conflicting-outputs
    ```
  </Accordion>
  
  <Accordion title="API connection issues">
    - Verify the backend server is running on `http://localhost:8787`
    - For Android emulator, use your computer's IP instead of `localhost`
    - Check firewall settings aren't blocking the connection
  </Accordion>
  
  <Accordion title="Flutter doctor issues">
    Run `flutter doctor -v` to see detailed diagnostic information and follow the recommendations to resolve any issues.
  </Accordion>
  
  <Accordion title="iOS build fails">
    - Ensure you have Xcode installed (macOS only)
    - Run `pod install` in the `ios/` directory
    - Try cleaning: `flutter clean && cd ios && pod deintegrate && pod install`
  </Accordion>
</AccordionGroup>
